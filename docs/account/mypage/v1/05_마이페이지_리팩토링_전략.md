# 마이페이지 리팩토링 상세 설계: View-Logic 분리

## 1. 목적
현재 `MyPage.tsx` 파일이 비대해짐에 따라 가독성이 떨어지고 유지보수가 어려워졌습니다. 이를 **View(UI)**와 **Logic(데이터 처리)**으로 분리하고, 하위 컴포넌트를 모듈화하여 구조를 개선합니다.

## 2. 폴더 구조 재구성

기존 `src/pages/MyPage.tsx` 단일 파일을 `src/pages/MyPage/` 폴더로 변경하고 다음과 같이 구성합니다.

```
src/pages/MyPage/
├── MyPage.tsx                  # [Main View] 껍데기 및 레이아웃 조립
├── useMyPage.ts                # [Logic Hook] 상태 관리 및 비즈니스 로직
└── components/                 # [Sub Components] 탭 및 모달 분리
    ├── Sidebar.tsx             # 좌측 프로필 정보 및 메뉴
    ├── OverviewTab.tsx         # '개요' 탭 컨텐츠
    ├── PostsTab.tsx            # '작성한 글' 탭 컨텐츠
    ├── CommentsTab.tsx         # '작성한 댓글' 탭 컨텐츠
    ├── BookmarksTab.tsx        # '찜한 과정' 탭 컨텐츠
    └── modals/                 # 마이페이지 전용 모달
        ├── EditProfileModal.tsx
        └── DeleteAccountModal.tsx
```

## 3. 파일별 상세 역할 및 구현 계획

### 3.1. `useMyPage.ts` (Logic)
*   **역할**: `MyPage`의 "두뇌" 역할.
*   **포함 내용**:
    *   `useAuthStore` 접근 (user 정보)
    *   `useState` (activeTab, modal open states)
    *   `useQuery` (프로필 정보 조회)
    *   `useMutation` (프로필 수정, 회원 탈퇴)
    *   `useForm` (프로필 수정 폼 핸들링) - *단, 폼 로직이 복잡하면 Modal 컴포넌트로 위임 가능*
    *   이벤트 핸들러 (`handleTabChange`, `handleLogout` 등)
*   **반환값**: View가 렌더링에 필요한 데이터와 함수들.

```typescript
// 예시
export const useMyPage = () => {
    const { user } = useAuthStore();
    const [activeTab, setActiveTab] = useState('overview');
    // ... queries & mutations ...

    return {
        user,
        activeTab,
        setActiveTab,
        // ...
    };
};
```

### 3.2. `MyPage.tsx` (Main View)
*   **역할**: `useMyPage` 훅을 사용하여 데이터를 받아오고, 하위 컴포넌트들을 배치(Layout)함.
*   **특징**: 로직 코드가 거의 없고 JSX 위주로 구성됨.

```tsx
// 예시
const MyPage = () => {
    const { user, activeTab, setActiveTab } = useMyPage();

    return (
        <div className="container">
            <Sidebar user={user} />
            <main>
                {activeTab === 'overview' && <OverviewTab />}
                {/* ... */}
            </main>
        </div>
    );
};
```

### 3.3. `components/` (Sub Views)
*   **Sidebar.tsx**: 프로필 요약 정보, 메뉴 버튼들.
*   **Tabs**: 각 탭의 내부 컨텐츠만 담당. `MyPage.tsx`에 있던 거대한 `switch-case` 문을 제거하고 각 파일로 분산.
*   **Modals**: `EditProfileModal`, `DeleteAccountModal` 등을 별도 파일로 분리하여 `MyPage.tsx`의 JSX 깊이를 줄임.

## 4. 마이그레이션 단계 (Step-by-Step)

1.  **폴더 생성**: `src/pages/MyPage/` 및 하위 폴더 생성.
2.  **하위 컴포넌트 분리**:
    *   `Sidebar` 분리.
    *   각 탭(`Overview`, `Posts` 등) 분리.
    *   모달(`EditProfile`, `DeleteAccount`) 분리.
3.  **Hook 분리**: `MyPage.tsx`에 있는 로직을 `useMyPage.ts`로 이동.
4.  **조립**: `MyPage.tsx`에서 Hook과 컴포넌트를 import하여 재조립.
5.  **검증**: 기능이 기존과 동일하게 동작하는지 확인.

## 5. 참고 사항
*   `EmailVerificationModal`과 `ChangePasswordModal`은 `src/components/auth/`에 있는 공용 컴포넌트를 그대로 재사용합니다.
*   Mock 데이터(`myPosts`, `myComments` 등)는 당분간 `useMyPage.ts` 내부에 두거나 별도 상수 파일(`constants.ts`)로 분리합니다.
