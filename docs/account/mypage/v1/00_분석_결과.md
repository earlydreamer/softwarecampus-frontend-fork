## ❌ **누락된 기능 (백엔드 API 있음)**

### 1️⃣ 프로필 조회 API 연동
**엔드포인트**: `GET /api/mypage/profile`

**현재 문제:**
- authStore 데이터만 표시 (로그인 시점 데이터)
- 페이지 마운트 시 최신 프로필 조회 안 함
- 서버에서 프로필 변경 시 반영 안 됨

**필요 작업:**
```typescript
// useEffect에서 프로필 조회
useEffect(() => {
  const fetchProfile = async () => {
    const profile = await getProfile();
    setUser(profile); // authStore 업데이트
  };
  fetchProfile();
}, []);
```

## ⚠️ **타입 불일치 문제**

### 백엔드 API vs 프론트엔드 타입 필드명 차이

| 필드 | 백엔드 API | 프론트엔드 타입 | 상태 |
|------|-----------|----------------|------|
| 승인 상태 | `approvalStatus` | `accountApproved` | ⚠️ 불일치 |

**영향 범위:**
```typescript
// MyPage.tsx Line 140, 148
user.accountApproved  // 백엔드는 approvalStatus

// types/index.ts
interface Account {
  accountApproved: ApprovalStatus;  // 백엔드와 필드명 다름
}
```

**해결 방법:**
1. **백엔드 맞추기 (권장)**: `accountApproved` → `approvalStatus`
2. **API 응답 변환**: DTO 변환 레이어에서 매핑
