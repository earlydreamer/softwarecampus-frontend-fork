# 마이페이지 종합 구현 요구사항 (프론트엔드 & 백엔드)

## 1. 개요
본 문서는 마이페이지의 **필수 기능(Core)**과 **확장 기능(Activity)**을 종합적으로 분석하여, 프론트엔드와 백엔드 간의 기능 격차(Gap)를 해소하고 구현 우선순위를 정의하기 위해 작성되었습니다.

## 2. 기능 분류 및 현황 분석

### 2.1. 필수 기능 (Core: 계정 관리)
사용자가 계정을 유지하고 관리하기 위해 반드시 필요한 기능입니다.

| 기능 | 프론트엔드 (UI) | 백엔드 (API) | 상태 및 필요 작업 |
| :--- | :--- | :--- | :--- |
| **프로필 조회** | ✅ 구현됨 (`MyPage.tsx`) | ✅ 있음 (`GET /profile`) | **[연동 대기]** 데이터 바인딩만 남음 |
| **프로필 수정** | ⚠️ 버튼만 존재 | ✅ 있음 (`PATCH /profile`) | **[UI 구현 필요]** 수정 모달 또는 페이지 개발 필요 |
| **비밀번호 변경** | ❌ **없음** | ❌ **없음** | **[신규 개발 필요]** 보안상 프로필 수정과 분리하여 구현 권장 |
| **회원 탈퇴** | ❌ **없음** | ✅ 있음 (`DELETE /account`) | **[UI 구현 필요]** 탈퇴 확인 팝업 및 버튼 추가 필요 |
| **프로필 사진** | ⚠️ UI만 있음 (아이콘) | ❌ **없음** | **[신규 개발 필요]** 이미지 업로드 API 및 DB 필드 추가 필요 |

커뮤니티 및 교육 플랫폼의 특성을 살린 대시보드 기능입니다.

| 기능 | 프론트엔드 (UI) | 백엔드 (API) | 상태 및 필요 작업 |
| :--- | :--- | :--- | :--- |
| **활동 통계 (KPI)** | ✅ 구현됨 (Mock) | ❌ **없음** | **[API 필요]** 작성 글/댓글 수, 총 조회수 등 |
| **내 게시글 목록** | ✅ 구현됨 (Mock) | ❌ **없음** | **[API 필요]** 페이징 처리된 내 글 목록 |
| **내 댓글 목록** | ✅ 구현됨 (Mock) | ❌ **없음** | **[API 필요]** 원글 제목(`boardTitle`) 포함 필수 |
| **찜한 강좌 목록** | ✅ 구현됨 (Mock) | ❌ **없음** | **[API 필요]** 찜한 강좌 상세 정보 |

## 3. 상세 요구사항 정의

### 3.1. [필수] 비밀번호 변경 (신규 API 필요)
현재 `03_mypage.md` 명세에 비밀번호 변경 API가 존재하지 않습니다. 보안을 위해 프로필 수정과 분리하여 별도 엔드포인트로 구현해야 합니다.

#### API 명세 제안
- **URL**: `/api/mypage/password`
- **Method**: `PUT`
- **인증**: 필요 (Bearer Token)
- **Request Body**:
  ```json
  {
    "currentPassword": "oldPassword123!",
    "newPassword": "newPassword123!",
    "confirmPassword": "newPassword123!"
  }
  ```
- **Validation**:
  - `currentPassword`: 현재 DB에 저장된 비밀번호와 일치해야 함.
  - `newPassword`: 비밀번호 복잡도 정책 준수 (8자 이상, 특수문자 포함 등).
  - `newPassword` != `currentPassword`: 현재 비밀번호와 달라야 함.
- **Response**: `200 OK` (성공 시)

### 3.2. [필수] 회원 탈퇴 (UI 추가)
*   **프론트엔드**: "정말로 탈퇴하시겠습니까?" 경고 모달 -> "탈퇴하기" 버튼 (빨간색).
*   **백엔드**: `DELETE /api/mypage/account` (이미 존재).
    *   Soft Delete 처리 확인 필요.

### 3.3. [확장] 활동 내역 API (신규 API 필요)
프론트엔드 대시보드 UI(`MyPage.tsx`)를 지원하기 위해 다음 API들이 필요합니다. 현재 백엔드에는 해당 기능을 지원하는 API가 없습니다.

#### 1. 활동 통계 조회
- **URL**: `/api/mypage/stats`
- **Method**: `GET`
- **Response**:
  ```json
  {
    "postCount": 15,      // 작성한 게시글 수
    "commentCount": 42,   // 작성한 댓글 수
    "bookmarkCount": 8,   // 찜한 강좌 수
    "totalViews": 1250    // 내 글의 총 조회수 (선택사항)
  }
  ```

#### 2. 내 게시글 목록 조회
- **URL**: `/api/mypage/posts`
- **Method**: `GET`
- **Query Params**: `page=0`, `size=5`
- **Response**:
  ```json
  {
    "content": [
      {
        "id": 101,
        "title": "스프링 부트 질문있습니다",
        "category": "Q&A",
        "views": 45,
        "commentCount": 2,
        "createdAt": "2025-11-28T10:00:00"
      }
    ],
    "totalPages": 3,
    "totalElements": 15
  }
  ```

#### 3. 내 댓글 목록 조회
- **URL**: `/api/mypage/comments`
- **Method**: `GET`
- **Query Params**: `page=0`, `size=5`
- **Response**:
  ```json
  {
    "content": [
      {
        "id": 501,
        "content": "저도 같은 생각입니다.",
        "boardId": 101,
        "boardTitle": "스프링 부트 질문있습니다", // 원글 제목 필수
        "createdAt": "2025-11-28T11:30:00"
      }
    ],
    "totalPages": 9,
    "totalElements": 42
  }
  ```

#### 4. 찜한 강좌 목록 조회
- **URL**: `/api/mypage/bookmarks`
- **Method**: `GET`
- **Query Params**: `page=0`, `size=5`
- **Response**:
  ```json
  {
    "content": [
      {
        "id": 201,
        "title": "자바 스프링 완전 정복",
        "instructor": "김강사",
        "rating": 4.8,
        "price": 55000,
        "thumbnailUrl": "/images/course/java.jpg"
      }
    ],
    "totalPages": 2,
    "totalElements": 8
  }
  ```

## 4. 구현 로드맵 (Priorities)

### Phase 1: 필수 기능 완성 (최우선)
1.  **타입 불일치 해결**: `approvalStatus` vs `accountApproved` 통일.
2.  **프로필 조회 연동**: `MyPage.tsx`에 실제 데이터 바인딩.
3.  **프로필 수정 UI 개발**: 모달 창 구현 및 `PATCH` API 연동.
4.  **회원 탈퇴 UI 개발**: 탈퇴 버튼 및 로직 구현.

### Phase 2: 보안 강화 (필수)
1.  **비밀번호 변경 기능**: 백엔드 API 추가 요청 및 프론트 UI 구현.

### Phase 3: 대시보드 고도화 (권장)
1.  **활동 내역 API 개발**: 백엔드에 통계/글/댓글/찜 API 구현 요청.
2.  **Mock Data 제거**: 프론트엔드 `MyPage.tsx`의 Mock 데이터를 실제 API로 교체.

## 5. 결론
현재 마이페이지는 **"보여주기식 UI(Mock)"** 상태입니다.
실제 서비스 가능한 수준이 되려면 **Phase 1(필수 기능)**과 **Phase 2(보안)**가 선행되어야 하며, **Phase 3(활동 내역)**는 백엔드 지원 일정에 맞춰 진행해야 합니다.
