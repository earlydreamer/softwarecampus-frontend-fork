## 🔧 구현 방식: URL Query Parameter

### 장점
- URL에 상태가 보여서 공유/북마크 가능
- 새로고침해도 유지됨
- 직관적이고 디버깅 쉬움

### URL 예시
| 현재 탭 | 글쓰기 URL |
|--------|-----------|
| 전체 | `/community/write` |
| 공지사항 | `/community/write?category=NOTICE` |
| 문의사항 | `/community/write?category=QUESTION` |
| 진로이야기 | `/community/write?category=COURSE_STORY` |
| 코딩이야기 | `/community/write?category=CODING_STORY` |

## 🔧 변경 코드

### CommunityPage.tsx
```tsx
// 기존
<Link to="/community/write" className="btn-primary flex items-center gap-2">

// 변경
<Link 
    to={`/community/write${currentCategory ? `?category=${currentCategory}` : ''}`}
    className="btn-primary flex items-center gap-2"
>
```

### CommunityWritePage.tsx
```tsx
import { useNavigate, useSearchParams } from 'react-router-dom';

const CommunityWritePage = () => {
    const [searchParams] = useSearchParams();
    
    // URL에서 카테고리 읽기, 유효하지 않으면 기본값 'CODING_STORY'
    const getInitialCategory = (): BoardCategory => {
        const urlCategory = searchParams.get('category') as BoardCategory;
        const validCategories: BoardCategory[] = ['NOTICE', 'QUESTION', 'COURSE_STORY', 'CODING_STORY'];
        return validCategories.includes(urlCategory) ? urlCategory : 'CODING_STORY';
    };
    
    const [category, setCategory] = useState<BoardCategory>(getInitialCategory());
    // ...
};
```

## ⚠️ 고려사항

1. **공지사항 카테고리**: 관리자만 선택 가능하도록 권한 체크 필요 (추후 구현)
2. **잘못된 카테고리**: URL에 유효하지 않은 카테고리 입력 시 기본값 `CODING_STORY` 사용
3. **기본값**: `QUESTION` → `CODING_STORY`로 변경 (가장 일반적인 게시글)
