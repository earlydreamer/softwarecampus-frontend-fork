# 프론트엔드 UI/UX 시나리오: 기관 등록 및 회원가입

## 1. 전체 흐름 (User Flow)

### Case A: 이미 등록된 기관의 소속원인 경우
1. **회원가입 페이지** 접속 (`/signup`)
2. **[기관회원]** 탭 선택
3. **"소속 기관 선택하기"** 버튼 클릭 → **기관 선택 모달** 오픈
4. 기관명 검색 → 목록에서 본인 소속 기관 선택
5. 나머지 회원정보(이메일, 이름 등) 입력
6. **"회원가입"** 버튼 클릭
   - API: `POST /api/auth/signup`
7. **완료 팝업(Alert/Modal) 표시**:
   - **제목**: 회원가입 신청 완료
   - **내용**: 
     > "회원가입 신청이 접수되었습니다.\n
     > 관리자 승인(1~2일 소요) 후 로그인이 가능합니다.\n
     > 승인 완료 시 이메일로 알려드립니다."
   - **버튼**: "로그인으로 이동"
8. **이동**: 로그인 페이지로 리다이렉트

### Case B: 기관을 새로 등록해야 하는 경우 (신규)
1. **회원가입 페이지** 접속 (`/signup`)
2. **[기관회원]** 탭 선택
3. **"소속 기관 선택하기"** 버튼 클릭 → **기관 선택 모달** 오픈
4. 기관명 검색 → 결과 없음
5. 모달 하단 **"기관 등록하기"** 버튼 클릭 → **등록 폼**으로 전환 (모달 내부)
6. **기관 정보 입력**:
   - 기관명, 사업자등록번호, 주소, 담당자 이메일
7. **증빙 서류 업로드** (필수):
   - 사업자등록증 또는 재직증명서 (PDF, 이미지)
8. **"등록 요청"** 클릭
   - API: `POST /academies` (Multipart)
   - **로딩 표시**: "기관 등록 요청 중입니다..." (스피너)
9. **완료 팝업(Alert/Modal) 표시**:
   - **제목**: 기관 등록 신청 완료
   - **내용**: 
     > "기관 등록 신청이 성공적으로 접수되었습니다.\n
     > 관리자 승인까지 약 1~2 영업일이 소요됩니다.\n
     > 승인 결과는 입력하신 이메일(**user@email.com**)로 발송됩니다.\n
     > 승인 완료 후 다시 회원가입을 진행해주세요."
   - **버튼**: "확인"
10. **회원가입 중단 및 이동**:
    - "확인" 클릭 시 모달 닫힘
    - 회원가입 페이지 초기화 또는 로그인 페이지로 이동 (사용자 선택 유도)

### Case C: 승인 완료 후 회원가입 (재방문)
1. (이메일 수신: "기관 등록이 승인되었습니다")
2. **회원가입 페이지** 재접속
3. **[기관회원]** 탭 → **기관 선택 모달**
4. 이제 검색 결과에 **내 기관이 표시됨**
5. 기관 선택 후 회원가입 진행

## 2. 주요 UI/UX 수정 사항

### 2.1 기관 선택 모달 (`AcademySelectModal.tsx`)
- **등록 폼 기능 강화**:
  - 현재 목업(Mock) 상태인 등록 기능을 실제 API와 연동
  - 파일 업로드 UI 개선 (드래그 앤 드롭, 미리보기 유지)
  - 필수 입력값 유효성 검사 (사업자번호 형식 등)
- **완료 피드백 강화 (Success Feedback)**:
  - 등록 성공 시 **별도의 성공 모달(Success Modal)** 또는 **SweetAlert** 스타일의 팝업 표시
  - 사용자가 "승인 대기" 상태임을 명확히 인지하고 이탈할 수 있도록 유도
  - 모달이 닫히면 상위 모달(기관 선택)도 함께 닫힘

### 2.2 회원가입 페이지 (`SignupPage.tsx`)
- **기관회원 탭 안내**:
  - "소속 기관이 검색되지 않으면 먼저 기관 등록을 신청해주세요." 문구 추가

## 3. 데이터 연동 계획

### API 호출
- **기관 목록 조회**: `GET /academies?approved=true` (검색어 필터링 필요)
- **기관 등록 신청**: `POST /academies` (FormData: json + files)

### 상태 관리
- **로딩 상태**: 파일 업로드 중 버튼 비활성화 및 스피너 표시
- **에러 처리**: 파일 크기 초과, 확장자 오류, 서버 에러 대응
