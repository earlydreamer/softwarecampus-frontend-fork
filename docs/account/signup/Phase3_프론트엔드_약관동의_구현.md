# Phase 3: 프론트엔드 약관 동의 구현

## 1. 개요
회원가입 시 서비스 이용약관, 개인정보 처리방침, 마케팅 정보 수신 동의를 받는 기능을 구현합니다.
백엔드 API는 이미 해당 필드(`termsAgreed`, `privacyAgreed`, `marketingAgreed`)를 수신하도록 업데이트되었습니다.

## 2. 작업 내용

### 2.1 타입 정의 업데이트 (`src/types/index.ts`)
`SignupFormData` 인터페이스에 약관 동의 관련 필드를 추가합니다.

```typescript
export interface SignupFormData {
    // ... 기존 필드
    termsAgreed: boolean;      // 이용약관 동의 (필수)
    privacyAgreed: boolean;    // 개인정보 수집 및 이용 동의 (필수)
    marketingAgreed: boolean;  // 마케팅 정보 수신 동의 (선택)
}
```

### 2.2 회원가입 페이지 UI 수정 (`src/pages/SignupPage.tsx`)
- 회원가입 폼 하단(가입하기 버튼 위)에 약관 동의 섹션을 추가합니다.
- "전체 동의" 체크박스와 개별 약관 체크박스를 구현합니다.
- 필수 약관 미동의 시 가입하기 버튼을 비활성화하거나 경고 메시지를 표시합니다.

### 2.3 API 연동
- `authService.signup` 호출 시 추가된 필드 값을 함께 전송합니다.

## 3. UI 상세 설계
- **전체 동의**: 모든 약관에 한 번에 동의할 수 있는 체크박스
- **이용약관 동의 (필수)**: [내용 보기] 링크 포함
- **개인정보 수집 및 이용 동의 (필수)**: [내용 보기] 링크 포함
- **마케팅 정보 수신 동의 (선택)**: [내용 보기] 링크 포함

## 4. 검증 계획
1. 모든 필수 약관 동의 시에만 회원가입 요청이 전송되는지 확인
2. 선택 약관 동의 여부가 백엔드에 올바르게 저장되는지 확인 (가입 후 DB 확인 필요)
3. 전체 동의 체크박스 동작 확인 (전체 선택/해제)
