# 필터 디자인 개선 보고서 (라벨 분리)

## 1. 문제 상황
- **사용자 피드백**: "레이블을 안에 넣지 말고 따로 분리해. 현재 선택된 항목이 뭔지 보여야 해."
- **기존 문제**: 드롭다운 버튼 안에 라벨이 포함되어 있어, 값을 선택하면 라벨("교육 대상" 등)이 사라지고 선택된 값("재직자" 등)만 보여, 현재 어떤 필터인지 명확하지 않을 수 있음.

## 2. 해결 방법
- **라벨 분리 (Label Separation)**:
  - 필터 라벨("교육 대상", "교육 방식")을 드롭다운 버튼 **상단**에 작은 텍스트로 분리하여 배치.
  - 버튼 내부에는 **현재 선택된 값**("전체", "재직자" 등)만 표시.
  
- **UI/UX 개선**:
  - 라벨을 상단에 배치함으로써, 사용자가 값을 선택하더라도 "이것이 어떤 필터인지" 항상 인지할 수 있음.
  - 버튼 스타일을 깔끔하게 유지하면서도 정보 전달력을 높임.
  - 드롭다운 메뉴의 너비를 충분히 확보(`min-w-[144px]`)하여 긴 텍스트도 잘리지 않도록 함.

## 3. 변경 코드 (`src/pages/CourseListPage.tsx`)

```tsx
// FilterDropdown 컴포넌트
<div className="flex flex-col gap-1.5">
    {/* 라벨을 버튼 밖으로 분리 */}
    <span className="text-xs font-semibold text-slate-500 ml-1">{label}</span>
    
    <div className="relative">
        <button className="...">
            {/* 버튼 안에는 선택된 값만 표시 */}
            <span className="truncate">{displayLabel}</span>
            <ChevronDown />
        </button>
        {/* ... 드롭다운 메뉴 ... */}
    </div>
</div>
```

## 4. 결과
- **명확성 향상**: 필터의 목적(라벨)과 현재 상태(값)가 동시에 명확하게 보임.
- **사용성 개선**: 폼 입력 필드와 유사한 익숙한 구조로 사용자가 쉽게 이해하고 조작할 수 있음.
