# 모바일 UX 개선 작업 보고서

## 1. 배경 및 목적
- **현황**: 모바일 화면에서 과정 목록과 커뮤니티 글이 세로로 길게 나열되어 스크롤 길이가 과도하게 길어짐.
- **문제점**: 사용자가 하단 콘텐츠(푸터 등)에 도달하기 어렵고, 정보 탐색 피로도가 높음.
- **목적**: 가로 스크롤(Carousel) UI를 도입하여 공간 효율성을 높이고 사용자 경험(UX)을 개선함.

## 2. 주요 변경 사항

### 2.1 공통 스타일 추가 (`src/index.css`)
- `hide-scrollbar` 유틸리티 클래스 추가
  - 스크롤 기능은 유지하되 시각적으로 스크롤바를 숨겨 깔끔한 UI 제공.

### 2.2 과정 목록 섹션 개선 (`src/components/home/CourseSection.tsx`)
- **변경 전**: `grid-cols-1`로 세로 나열.
- **변경 후**:
  - 모바일(`sm` 미만): `flex`, `overflow-x-auto`를 사용하여 가로 스크롤 적용.
  - 데스크탑(`sm` 이상): 기존 `grid` 레이아웃 유지.
  - `snap-x`, `snap-center` 속성을 사용하여 스크롤 시 카드가 중앙에 정렬되도록 개선.

### 2.3 커뮤니티 섹션 개선 (`src/components/home/CommunitySection.tsx`)
- **변경 전**: `grid-cols-1`로 세로 나열.
- **변경 후**:
  - 모바일(`md` 미만): 가로 스크롤 적용.
  - 데스크탑(`md` 이상): 기존 `grid` 레이아웃 유지.
  - 카드 너비를 화면 너비의 85%(`w-[85vw]`)로 설정하여 다음 카드가 살짝 보이도록 유도(Affordance 제공).

## 3. 기대 효과
- 모바일 메인 페이지의 전체 길이가 단축됨.
- 사용자가 좌우 스와이프 제스처를 통해 더 빠르고 직관적으로 콘텐츠를 탐색 가능.
- 앱과 유사한 네이티브한 사용성 제공.
