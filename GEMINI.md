# Gemini 전용 지침

> ⚙️ 이 모델은 `AGENTS.md`를 기본 행동 지침으로 참조합니다. 아래 항목은 Gemini 사용 시 추가로 적용되는 운영 지침입니다.

## 역할과 범위
- **프론트엔드(React + TypeScript) 기여를 우선 지원**하며, 문서/리뷰/리팩터링을 포함합니다.
- 답변은 항상 **한국어로 간결하고 실행 가능하게** 작성합니다.
- 사용자의 의도를 정확히 파악하고, 불명확한 경우 먼저 확인합니다.

## 우선순위
1. **정확성**: 잘못된 정보보다 "모름"이 낫습니다.
2. **레이어 규칙 준수**: 컴포넌트 분리, Props 전달, 상태 관리 계층 유지
3. **최소 변경**: 요청된 범위만 수정하고, 불필요한 리팩터링 지양
4. **테스트 동반**: 중요 로직 변경 시 테스트 작성 제안

## 코드 작성 원칙
- **TypeScript**: 명시적 타입 정의, `any` 사용 금지
- **React**: 함수형 컴포넌트, Hooks 우선 사용
- **보안**: 사용자 입력은 항상 검증/살균 처리
- **접근성**: ARIA 속성, 시맨틱 HTML 적용
- **성능**: 불필요한 리렌더링 방지(memo, useMemo, useCallback)

## 기술 스택 참조
- 상세한 기술 스택은 `FRONTEND-STACKS.md` 참조
- 디자인 시스템은 `DESIGN-SYSTEM.md` 참조
- 백엔드 연동은 `BACKEND_FRONTEND_MAPPING.md` 참조

## 응답 형식
1. 변경 사항 요약 (간결하게)
2. 코드 수정 (필요시)
3. 주의사항 또는 추가 제안 (선택)
